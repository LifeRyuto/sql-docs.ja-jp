# <a name="odbc-glossary"></a>ODBC の用語集
## <a name="a"></a>A  
 **アクセスプラン**  
 SQL ステートメントを実行するためにデータベース エンジンによって生成されたプラン。 C などの第 3 世代言語からコンパイルされた実行可能コードと同等です。  
  
 **集計関数 (aggregate function)**  
 **GROUP BY**句および**HAVING**句で使用される、値のグループから単一の値を生成する関数。 集計関数には **、AVG**、**カウント**、**最大**、**および合計**が**含まれます。** *セット関数*とも呼ばれます。 *スカラー関数も参照してください*。  
  
 **Ansi**  
 アメリカ国立標準研究所 ODBC API は、ANSI 呼び出しレベル のインターフェイスに基づいています。  
  
 **APD**  
 アプリケーション・パラメーター記述子 (APD)*を参照してください*。  
  
 **API**  
 アプリケーション プログラミング インターフェイス。 アプリケーションが、低レベルのサービスを要求および実行するために使用する一連のルーチン。 ODBC API は、ODBC 関数で構成されています。  
  
 **application**  
 ODBC API 内の関数を呼び出す実行可能プログラム。  
  
 **アプリケーション・パラメーター記述子 (APD)**  
 アプリケーションで指定された変換の前に SQL ステートメントで使用される動的パラメーターを記述する記述子。  
  
 **アプリケーション行記述子 (ARD)**  
 アプリケーションのバッファー内の列メタデータとデータを表す記述子。  
  
 **Ard**  
 アプリケーション行記述子 (ARD)*を参照してください*。  
  
 **自動コミット モード**  
 トランザクションが実行された直後にコミットされるトランザクションコミット モード。  
  
## <a name="b"></a>B  
 **動作の変更**  
 ODBC *3.x*の動作から ODBC *2.x*の動作への変更、またはその逆の変更。 SQL_ATTR_ODBC_VERSION環境属性の変更によって発生します。  
  
 **バイナリ ラージ オブジェクト (BLOB)**  
 255 など、特定のバイト数を超えるバイナリ データ。 通常ははるかに長いです。 このようなデータは、通常、データ ソースに対して部分的に送信および取得されます。 *長いデータ*とも呼ばれます。  
  
 **バインディング**  
 動詞として、結果セット内の列、または SQL ステートメント内のパラメーターをアプリケーション変数に関連付ける動作。 名詞として、関連付け。  
  
 **バインド オフセット**  
 すべてのバインドされた列またはパラメーター データのデータ バッファー アドレスと長さ/インジケーター バッファー アドレスに追加される値。  
  
 **ブロック カーソル (block cursor)**  
 一度に複数のデータ行をフェッチできるカーソル。  
  
 **バッファー**  
 アプリケーションとドライバーの間でデータを渡すために使用されるアプリケーション メモリの一部。 バッファーは、*データ バッファー*と*データ長バッファー*のペアで入ってくることがよくあります。  
  
 **byte**  
 8ビットまたは1オクテット。 「オクテット *」も参照してください*。  
  
## <a name="c"></a>C  
 **C データ型**  
 C プログラム (この場合はアプリケーション) 内の変数のデータ型。  
  
 **カタログ**  
 データベースの形状を記述するデータベース内のシステム テーブルのセット。 *スキーマ*辞書または*データディクショナリ*とも呼ばれます。  
  
 **カタログ機能**  
 データベースのカタログから情報を取得するために使用される ODBC 関数。  
  
 **CLI**  
 *参照*Api。  
  
 **クライアント/サーバー**  
 1 つ以上のクライアントがサーバーを介してデータにアクセスするデータベース アクセス方法。 クライアントは通常、サーバーがデータベース アクセスを制御している間にユーザー インターフェイスを実装します。  
  
 **列**  
 1 行の情報の単一項目のコンテナー。 *フィールド*とも呼ばれます。  
  
 **コミット**  
 トランザクションの変更を永続的に行う。  
  
 **並行 処理**  
 複数のトランザクションが同時に同じデータにアクセスする機能。  
  
 **適合レベル**  
 ドライバーまたはデータ ソースでサポートされる個別の機能のセット。 ODBC は、API 準拠レベルと SQL 準拠レベルを定義します。  
  
 **connection**  
 ドライバーとデータ ソースの特定のインスタンス。  
  
 **接続の参照**  
 ネットワークで接続先のデータ ソースを検索します。 接続の参照には、いくつかの手順が含まれる場合があります。 たとえば、ユーザーはまずネットワークでサーバーを参照し、次に特定のサーバーでデータベースを参照します。  
  
 **接続ハンドル**  
 接続に関する情報を格納するデータ構造体へのハンドル。  
  
 **現在の行**  
 現在カーソルが指している行。 位置付け操作は、現在の行に対して動作します。  
  
 **カーソル**  
 アプリケーションにデータ行を返すソフトウェア。 おそらくコンピュータ端末の点滅カーソルにちなんで名付けられました。カーソルが画面上の現在位置を示すのと同様に、結果セット上のカーソルは結果セット内の現在位置を示します。  
  
## <a name="d"></a>D  
 **データ バッファ**  
 データを渡すために使用されるバッファー。 データ バッファに関連付けられるのは、*データ長バッファ*です。  
  
 **データ辞書 (data dictionary)**  
 *カタログを参照してください*。  
  
 **データ長バッファー**  
 対応するデータ バッファの値の長さを渡すために使用される*バッファ*。 データ長バッファーは、データ値が NULL で終わるかどうかなどの標識を保管するためにも使用されます。  
  
 **データ ソース**  
 ユーザーがアクセスするデータと、関連するオペレーティング システム、DBMS、およびネットワーク プラットフォーム (存在する場合)。  
  
 **データ型**  
 データの一部の型。 ODBC は C および SQL データ型を定義します。 *「タイプインジケーター」も参照してください*。  
  
 **実行時データ列**  
 **SQLSetPos**が呼び出された後にデータが送信される列。 データが行セット バッファーに配置されるのではなく、実行時に送信されるため、名前が付けられます。 長いデータは、通常、実行時に一部に送信されます。  
  
 **実行時データパラメーター**  
 **SQL 実行**または**SQLExecDirect**が呼び出された後にデータが送信されるパラメーター。 パラメータ バッファに格納されるのではなく、SQL ステートメントの実行時にデータが送信されるため、名前が付けられます。 長いデータは、通常、実行時に一部に送信されます。  
  
 **データベース**  
 DBMS 内のデータの個別のコレクション。 また、DBMS。  
  
 **データベース エンジン**  
 SQL ステートメントを解析して実行し、物理データにアクセスする DBMS 内のソフトウェア。  
  
 **DBMS (DBMS)**  
 データベース管理システム。 物理的なデータベースとユーザー間のソフトウェアの層。 DBMS はデータベースへのアクセス権をすべて管理します。  
  
 **DBMS ベースのドライバ**  
 スタンドアロン データベース エンジンを介して物理データにアクセスするドライバー。  
  
 **DDL (DDL)**  
 データ定義言語。 操作とは対照的にデータを定義する SQL のステートメント。 たとえば、**テーブルの作成**、**インデックスの作成**、**許可**、**取り消**しなどです。  
  
 **区切られた識別子 (delimited identifier)**  
 特殊文字を含めたり、キーワードに一致させるため識別子の引用符で囲まれた識別子 (引用符で囲まれた識別子とも呼ばれます)。  
  
 **記述子**  
 列データまたは動的パラメーターに関する情報を保持するデータ構造。 記述子の物理表現が定義されていません。アプリケーションは、記述子ハンドルを使用して ODBC 関数を呼び出すことによって、そのフィールドを操作することによってのみ、記述子に直接アクセスできます。  
  
 **デスクトップ データベース**  
 パーソナル コンピュータ上で動作するように設計された DBMS。 一般に、これらの DBMS はスタンドアロンのデータベース エンジンを提供せず、ファイル ベースのドライバーを通じてアクセスする必要があります。 これらのドライバのエンジンは、一般的に SQL とトランザクションのサポートを減らしています。 たとえば、dBASE、パラドックス、ブトリエブ、マイクロソフト®FoxPro®。  
  
 **診断**  
 特定のハンドルを使用した、呼び出された最後の関数に関する診断情報を含むレコード。 診断レコードは、環境、接続、ステートメント、および記述子ハンドルに関連付けられます。  
  
 **DML**  
 データ操作言語。 定義とは対照的にデータを操作する SQL のステートメント。 たとえば、**挿入**、**更新**、**削除**、 および**選択**などです。  
  
 **ドライバー**  
 ODBC API の関数を公開するルーチン ライブラリ。 ドライバは、単一の DBMS に固有です。  
  
 **ドライバー マネージャー**  
 アプリケーションのドライバーへのアクセスを管理するルーチン ライブラリ。 ドライバー マネージャーは、読み込みとアンロード (またはに接続して、接続と切断) ドライバーと ODBC 関数への呼び出しを正しいドライバーに渡します。  
  
 **ドライバ セットアップ DLL**  
 ドライバ固有のインストールおよび構成機能を含む DLL。  
  
 **動的カーソル (dynamic cursor)**  
 結果セット内で更新、削除、または挿入された行を検出できるスクロール可能なカーソル。  
  
 **動的 SQL**  
 SQL ステートメントが実行時に作成およびコンパイルされる組み込み SQL のタイプ。 静的 SQL*も参照してください*。  
  
## <a name="e"></a>E  
 **組み込み SQL**  
 別の言語で記述されたプログラムに直接組み込まれている SQL ステートメント (COBOL や C など) ODBC は組み込み SQL を使用しません。 静的 SQL*および*動的 SQL*も参照してください*。  
  
 **環境**  
 データにアクセスするグローバル コンテキスト。環境に関連付けられている情報は、その環境内のすべての接続のリストなど、本質的にグローバルな情報です。  
  
 **環境ハンドル**  
 環境に関する情報を格納するデータ構造体へのハンドル。  
  
 **エスケープ句**  
 SQL ステートメント内の句。  
  
 **実行**  
 SQL ステートメントを実行します。  
  
## <a name="f"></a>F  
 **脂肪カーソル**  
 *ブロックカーソルを参照してください*。  
  
 **フェッチ**  
 結果セットから 1 つ以上の行を取得します。  
  
 **フィールド**  
 *列を参照してください*。  
  
 **ファイル ベースのドライバ**  
 物理データに直接アクセスするドライバー。 この場合、ドライバーにはデータベース エンジンが含まれ、ドライバーとデータ ソースの両方として機能します。  
  
 **ファイル データ ソース**  
 接続情報が .dsn ファイルに格納されるデータ ソース。  
  
 **外部キー (foreign key)**  
 別のテーブルの主キーと一致するテーブル内の列。  
  
 **前方専用カーソル**  
 結果セットを順方向に移動することしかできないカーソルで、通常は一度に 1 行しかフェッチできません。 ほとんどのリレーショナル データベースでは、前方専用カーソルのみがサポートされます。  
  
## <a name="h"></a>H  
 **処理**  
 ファイルやデータ構造などを一意に識別する値。 ハンドルは、それらを作成して使用するソフトウェアにのみ意味がありますが、物事を識別するために他のソフトウェアによって渡されます。 ODBC は、環境、接続、ステートメント、および記述子のハンドルを定義します。  
  
## <a name="i"></a>I  
 **実装パラメータ記述子 (IPD)**  
 アプリケーションで指定された変換後に SQL ステートメントで使用される動的パラメーターを記述する記述子。  
  
 **実装行記述子 (IRD)**  
 アプリケーションで指定された変換の前に、データ行を記述する記述子。  
  
 **インストーラ DLL**  
 ODBC コンポーネントをインストールし、データ ソースを構成する DLL。  
  
 **保全性拡張機能**  
 データベースの整合性を維持するために設計された SQL のサブセット。  
  
 **インターフェイス準拠レベル**  
 ドライバーでサポートされている ODBC 3.7 インターフェイスのレベル。コア、レベル 1、レベル 2 を指定できます。  
  
 **相互 運用 性**  
 異なる DBMS 内のデータにアクセスするときに、1 つのアプリケーションが同じコードを使用する機能。  
  
 **IPD**  
 *参照*実装パラメータ記述子 (IPD)。  
  
 **Ird**  
 *参照*実装行記述子 (IRD)。  
  
 **ISO/IEC**  
 国際標準化機構/国際電気技術委員会 ODBC API は、ISO/IEC のコールレベルインタフェースに基づいています。  
  
## <a name="j"></a>J  
 **join**  
 指定した列の値を照合して、複数のテーブルの行をリンクするリレーショナル データベースの操作。  
  
## <a name="k"></a>K  
 **key**  
 値が行を識別する列。 外部キー*と*主キー*も参照してください*。  
  
 **キーセット**  
 行を再フェッチするために混合カーソルまたはキーセットドリブン カーソルによって使用されるキーのセット。  
  
 **キーセット ドリブン カーソル (keyset-driven cursor)**  
 キーセットを使用して更新された行と削除された行を検出するスクロール可能なカーソル。  
  
## <a name="l"></a>L  
 **リテラル**  
 SQL ステートメント内の実際のデータ値の文字表現。  
  
 **ロック**  
 DBMS がマルチユーザー環境の行へのアクセスを制限するプロセス。 DBMS は通常、行または行またはページがロックされていることを示す行を含む物理ページにビットを設定します。  
  
 **長いデータ**  
 255 バイトや文字など、特定の長さ以上のバイナリ データまたは文字データ。 通常ははるかに長いです。 このようなデータは、通常、データ ソースに対して部分的に送信および取得されます。 *BLOB*または*CLOB*とも呼ばれます。  
  
## <a name="m"></a>M  
 **マシン データ ソース**  
 接続情報がシステムに格納されるデータ ソース (レジストリなど)。  
  
 **手動コミット モード**  
 トランザクションを明示的にコミットする必要があるトランザクション コミット モード**SQLTransact**を呼び出します。  
  
 **metadata**  
 SQL ステートメント内のパラメーター、または結果セット内の列を記述するデータ。 たとえば、データ型、バイト長、およびパラメーターの精度。  
  
 **多層ドライバー**  
 *参照*DBMS ベースのドライバ。  
  
## <a name="n"></a>N  
 **NULL 値**  
 値が明示的に割り当てられていない。 特に、NULL 値はゼロまたはブランクとは異なります。  
  
## <a name="o"></a>O  
 **オクテット**  
 8 ビットまたは 1 バイト。 *バイトも参照してください*。  
  
 **オクテットの長さ**  
 バッファーまたはバッファーに含まれるデータのオクテットの長さ。  
  
 **ODBC**  
 データベース接続を開きます。 アプリケーションがデータ ソース内のデータにアクセスできるルーチンの標準セットを定義する API の仕様。  
  
 **ODBC アドミニストレーター**  
 インストーラー DLL を呼び出してデータ ソースを構成する実行可能プログラム。  
  
 [グループを開く]  
 標準を公開する会社。 特に、SQL アクセス グループ (SAG) 標準を公開します。  
  
 **オプティミスティック同時実行制御**  
 行がロックされていない同時実行性を高める戦略。 代わりに、更新または削除される前に、カーソルは、それらが最後に読み取られた後に変更されているかどうかを確認します。 その場合、更新または削除は失敗します。 ペシミスティック同時実行制御*も参照してください*。  
  
 **外部結合**  
 一致する行と一致しない行の両方が返される結合。 一致しない行の一致しないテーブルのすべての列の値は NULL に設定されます。  
  
 **所有者**  
 テーブルの所有者。  
  
## <a name="p"></a>P  
 **パラメーター**  
 パラメータ マーカーまたは疑問符 (?) でマークされた SQL ステートメント内の変数。 パラメーターは、アプリケーション変数にバインドされ、ステートメントの実行時に取得される値が返されます。  
  
 **パラメータ記述子**  
 アプリケーションで指定された変換 (アプリケーション・パラメーター記述子、または APD) の前、またはアプリケーションで指定された変換 (実装パラメーター記述子、または IPD) の後に、SQL ステートメントで使用される実行時パラメーターを記述する記述子。  
  
 **パラメータ操作配列**  
 **SQLExecDirect**または**SQLExecute**操作で、対応するパラメーターを無視することを示すためにアプリケーションが設定できる値を含む配列。  
  
 **パラメータステータス配列**  
 **SQLExecDirect**または**SQL 実行**の呼び出し後のパラメーターの状態を格納する配列。  
  
 **ペシミスティック コンカレンシー**  
 他のトランザクションが変更できないように行がロックされる、直列化可能性を実装するための戦略。 オプティミスティック同時実行制御*と*直列性*も参照してください*。  
  
 **位置付け操作**  
 現在の行に対して動作する操作。 たとえば、位置指定更新ステートメントと削除ステートメント **、SQLGetData**、および**SQLSetPos を使用します**。  
  
 **位置指定更新ステートメント**  
 現在の行の値を更新するために使用される SQL ステートメント。  
  
 **位置指定削除ステートメント**  
 現在の行を削除するために使用される SQL ステートメント。  
  
 **準備**  
 SQL ステートメントをコンパイルします。 アクセス・プランは、SQL ステートメントを準備することによって作成されます。  
  
 **主キー**  
 テーブル内の行を一意に識別する列。  
  
 **手順**  
 データベース内の名前付きオブジェクトとして格納されている 1 つ以上のプリコンパイル済み SQL ステートメントのグループ。  
  
 **プロシージャ列**  
 プロシージャ呼び出しの引数、プロシージャによって返される値、またはプロシージャによって作成された結果セット内の列。  
  
## <a name="q"></a>Q  
 **修飾子**  
 1 つ以上のテーブルを含むデータベース。  
  
 **query**  
 SQL ステートメント。 **SELECT**ステートメントを意味する場合があります。  
  
 **引用符で囲まれた識別子 (quoted identifier)**  
 特殊文字を含めたり、キーワードに一致させるため、識別子の引用符で囲まれた識別子 (SQL-92 では区切り識別子としても知られています)。  
  
## <a name="r"></a>R  
 **基数**  
 数値システムのベース。 通常は2または10。  
  
 **レコード**  
 *行を参照してください*。  
  
 **結果セット**  
 **SELECT**ステートメントを実行して作成された行のセット。  
  
 **リターン コード**  
 ODBC 関数によって返される値。  
  
 **ロールバック**  
 トランザクションによって変更された値を元の状態に戻します。  
  
 **行**  
 特定のエンティティを記述する関連列のセット。 *レコード*とも呼ばれます。  
  
 **行記述子**  
 アプリケーションで指定された変換 (実装行記述子、IRD) の前、またはアプリケーションで指定された変換後 (アプリケーション行記述子または ARD) の後に、結果セットの列を記述する記述子。  
  
 **行操作配列**  
 **SQLSetPos**操作で対応する行を無視することを示すためにアプリケーションが設定できる値を含む配列。  
  
 **行の状態の配列**  
 **SQL フェッチ、SQL フェッチ****スクロール**、または**SQLSetPos**への呼び出し後の行の状態を格納する配列。  
  
 **行**  
 ブロック カーソルによって 1 回のフェッチで返される行のセット。  
  
 **行セット バッファー**  
 結果セットの列にバインドされ、行セット全体のデータが返されるバッファー。  
  
## <a name="s"></a>S  
 **サグ**  
 *参照*SQL アクセス グループ (SAG)  
  
 **スカラー機能**  
 単一の値から単一の値を生成する関数。 たとえば、文字データの大文字と小文字を変更する関数です。  
  
 **schema**  
 *カタログを参照してください*。  
  
 **スクロール可能なカーソル**  
 結果セットを前後に移動できるカーソル。  
  
 **シリアライズ可能性**  
 2 つのトランザクションを同時に実行すると、それらのトランザクションの逐次実行と同じ結果が生成されるかどうか。 データベースの整合性を維持するには、シリアル化可能なトランザクションが必要です。  
  
 **サーバー データベース**  
 クライアント/サーバ環境で実行するように設計された DBMS。 これらの DBMS は、SQL およびトランザクションに対して豊富なサポートを提供するスタンドアロン のデータベース エンジンを提供します。 これらは、DBMS ベースのドライバを通じてアクセスされます。 たとえば、Oracle、インフォーミックス、DB/2、またはマイクロソフト® SQL Server などです。  
  
 **機能を設定する**  
 *集計関数を参照してください*。  
  
 **セットアップ DLL**  
 ドライバーセットアップ DLL*と*トランスレータセットアップ DLL*を参照してください*。  
  
 **単層ドライバー**  
 ファイルベースのドライバ*を参照してください*。  
  
 **SQL**  
 構造化照会言語。 リレーショナル データベースがデータのクエリ、更新、および管理に使用する言語。  
  
 **SQL アクセス グループ (SAG)**  
 SQL Dbms に関係する企業の業界コンソーシアム。 オープングループのコールレベルインタフェースは、最初にSQLアクセスグループが行った作業に基づいています。  
  
 **SQL 準拠レベル**  
 ドライバーでサポートされている SQL-92 文法のレベル。入力、FIPS 移行、中間、または完全です。  
  
 **SQL データ型**  
 データ ソースに格納されている列またはパラメーターのデータ型。  
  
 **SQLSTATE**  
 特定のエラーを示す 5 文字の値。  
  
 **SQL ステートメント**  
 キーワードで始まり、実行するアクションを完全に記述する SQL の完全なフレーズ。 たとえば、[* から注文] を選択します。 SQL ステートメントとステートメントを混同しないでください。  
  
 **状態**  
 項目の明確な条件。 たとえば、接続には、未割り当て、割り当て済み、接続、および必要なデータを含む 7 つの状態があります。 特定の操作は、項目が特定の状態にある場合にのみ実行できます。 たとえば、接続が割り当てられた状態の場合にのみ解放され、接続状態の場合などは解放できません。  
  
 **状態遷移**  
 ある状態から別の状態への項目の移動。 ODBC では、環境、接続、およびステートメントに対して厳密な状態遷移を定義します。  
  
 **ステートメント**  
 SQL ステートメントに関連するすべての情報のコンテナー。 ステートメントを SQL ステートメントと混同しないでください。  
  
 **ステートメント ハンドル**  
 ステートメントに関する情報を格納するデータ構造体へのハンドル。  
  
 **静的カーソル (static cursor)**  
 結果セット内の更新、削除、または挿入を検出できないスクロール可能なカーソル。 通常は、結果セットのコピーを作成することによって実装されます。  
  
 **静的 SQL**  
 プログラムの残りの部分をコンパイルするときに SQL ステートメントをハードコーディングし、コンパイルする組み込み SQL のタイプ。 動的 SQL*も参照してください*。  
  
 **ストアド プロシージャ (stored procedure)**  
 *手順を参照してください*。  
  
## <a name="t"></a>T  
 **テーブル**  
 行のコレクション。  
  
 **サンク**  
 16 ビットのアプリケーションが 32 ビット ODBC ドライバーで使用される場合、16 ビット アドレスから 32 ビット アドレスへの変換、またはその逆の変換。  
  
 **トランザクション**  
 作業のアトミック単位。 トランザクションでの作業は全体として完了する必要があります。トランザクションの一部が失敗すると、トランザクション全体が失敗します。  
  
 **トランザクション分離**  
 1 つのトランザクションを他のすべてのトランザクションの影響から分離する行為。  
  
 **トランザクション分離レベル**  
 トランザクションがどの程度分離されているかを示す尺度。 トランザクション分離レベルには、コミットされていない、コミットされた読み取り、反復可能読み取り、シリアル化可能、およびバージョン管理の 5 つのレベルがあります。  
  
 **トランスレータ DLL**  
 ある文字セットから別の文字セットにデータを変換するために使用される DLL。  
  
 **トランスレータ セットアップ DLL**  
 トランスレータ固有のインストールおよび構成関数を含む DLL。  
  
 **2 フェーズ コミット (two-phase commit)**  
 分散トランザクションを 2 つのフェーズでコミットするプロセス。 最初のフェーズでは、トランザクション プロセッサは、トランザクションのすべての部分がコミット可能かどうかをチェックします。 第 2 フェーズでは、トランザクションのすべての部分がコミットされます。 トランザクションのいずれかの部分が、コミットできないことを最初のフェーズで示している場合、第 2 フェーズは発生しません。 ODBC では、2 フェーズ コミットはサポートされていません。  
  
 **タイプインジケータ**  
 アプリケーション変数、パラメーター、または列のデータ型を示すために ODBC 関数に渡されるか、ODBC 関数から返される整数値。 ODBC は、C および SQL の両方のデータ型の型標識を定義します。  
  
## <a name="v"></a>V  
 **ビュー**  
 1 つ以上のテーブルのデータを参照する別の方法。 ビューは通常、1 つ以上のテーブルの列のサブセットとして作成されます。 ODBC では、ビューは一般にテーブルと同等です。
