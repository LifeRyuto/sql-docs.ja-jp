# <a name="oracle-software-patches"></a>Oracle ソフトウェアの修正プログラム
> [!IMPORTANT]  
>  この機能は、今後のバージョンの Windows で削除される予定です。 新規の開発作業ではこの機能を使用しないようにし、現在この機能を使用しているアプリケーションは修正することを検討してください。 代わりに、Oracle によって提供される ODBC ドライバーを使用します。  
  
 Oracle サーバー製品とそのクライアント コンポーネントの修正プログラムは、Oracle 用 Microsoft ODBC ドライバ、Oracle 用 MICROSOFT OLE DB プロバイダ、インターネット インフォメーション サービス (IIS) 、コンポーネント サービス (Windows NT を使用している場合はマイクロソフト トランザクション サーバー) など、マイクロソフトの製品およびテクノロジを適切に機能させる必要があります。  
  
> [!NOTE]  
>  Oracle FTPサイトは変更される可能性があるため、以下の手順は完全に正確ではない可能性があります。  
  
### <a name="to-download-the-oracle-software-patches"></a>Oracle ソフトウェアパッチをダウンロードするには  
  
1.  oracle-ftp.oracle.comのパブリック FTP サイトに接続します。 ユーザー ID は "匿名" で、パスワードは電子メール アドレスです。  
  
2.  次のディレクトリに移動します: /サーバー/wgt_tech/サーバー/ウィンドウズ。  
  
3.  Windows 95、Windows 98、および Windows NT/2000 に最も関連するパッチをダウンロードするには、Oracle のバージョンのサブディレクトリに移動します - 7.3 または 8.0。 2 つのサブディレクトリは/73patchsets と /80 パッチセットです。  
  
4.  Oracle のネットワーク・テクノロジー(SQL*Net または Net8)のパッチをダウンロードするには、次のディレクトリー /network に移動します。  
  
 Web ブラウザからこの FTP サイトにアクセスできない場合があります。 問題が発生した場合は、"従来の" FTP クライアントを使用するか、DOS コマンド プロンプトを使用してください。  
  
> [!NOTE]  
>  Oracle は、現在のバージョンのバグを修正し、ソフトウェア パッチを使用して以前のバージョンに修正するため、利用可能な最新のパッチをダウンロードすることをお勧めします。 これは、Oracle サーバー クライアント コンポーネントに特に当てはまります。 これらのパッチのインストールについてご質問がある場合は、Oracleサポートにお問い合わせください。
